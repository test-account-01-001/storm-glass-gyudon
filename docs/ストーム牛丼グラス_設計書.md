## 1. システム概要

ストームグラス（気温変化で結晶が増えたり溶けたりする、昔ながらの天気占いガラス）の挙動を「牛丼」で再解釈した待受風Webアプリ。

画面下部に丼（どんぶり）が常に鎮座し、**その日の天気と気温に応じて、具材（肉・米・玉ねぎ等）が画面全体に浮遊・沈殿・白濁・枝状拡散する**。背景も天気・時間帯で変化する。スマホでフルスクリーン表示して待受として使う。

- 見た目: 3DCG（Three.js）。リアルな質感の具材が夜景の中を漂う
- 天気: 現在地から自動取得（Open-Meteo API・無料・APIキー不要）
- 公開: GitHub Pages
- 待受方式: PWA＋フルスクリーン＋時計＋スリープ防止（Wake Lock）

### 本物のストームグラスの仕様（変換の元ネタ）

ストームグラスは樟脳入りアルコール溶液で、**気温が下がると結晶が増え、上がると溶けて澄む**。伝統的な読み方（澄む=晴れ、白濁=雨や雪の前兆、点状=霧、星形=雷雨、底に沈殿=霜、糸状=風）を具材の動きに翻訳する。
※実物は科学的な予測精度のない「お守り」的道具。本アプリは実際の天気データを使って「読み方が当たっている世界」を演出する。

## 3. 処理フロー（入力→出力）

```
STEP1: ページを開く → ローディング表示（「天気を取得中…少々お待ちください」）
STEP2: 位置情報の許可を求める
        ├─ 許可 → 現在地の緯度経度＋地名（逆ジオコーディング）
        └─ 拒否/失敗 → デフォルト地点（東京）で続行 ※地点名に明示
STEP3: Open-Meteo APIから現在の天気を取得
        （天気コード・気温・体感温度・湿度・風速・降水確率）
STEP4: 変換テーブルで天気→「変化パターン」＋牛丼パラメータに変換
STEP5: Three.jsで丼＋具材を3D描画、背景を天気・時間帯に合わせて描画
STEP6: 時計・日付・天気情報パネル・操作ボタンをオーバーレイ表示
STEP7: 30分ごとに天気を再取得して自動更新（「天気更新」ボタンで手動更新も可）
        オフライン時は前回取得値で継続＋「通信に失敗しました もう一度お試しください」表示
```

## 4. 天気→牛丼 変換仕様（このアプリの心臓部）

### 4-1. 入力データ（Open-Meteoから取得）

| 項目 | APIフィールド | 用途 |
|---|---|---|
| 天気コード | `weather_code`（WMO規格） | 変化パターンの決定 |
| 気温（℃） | `temperature_2m` | 米の量・暑い日/寒い日判定 |
| 体感温度（℃） | `apparent_temperature` | 情報パネル表示（Feels like） |
| 湿度（%） | `relative_humidity_2m` | 白濁の補正・表示 |
| 風速（m/s） | `wind_speed_10m` | 枝状拡散・横流れ・表示 |
| 降水確率（%） | `precipitation_probability` | 情報パネル表示 |

### 4-2. 変化パターン（6種類）＝UI指示書Aセクション

ストームグラスの状態をそのまま具材の振る舞いに翻訳した6パターン。画面下部に現在のパターン名を表示する（例: `Storm: 牛丼乱気流 / Pattern: 浮遊型`）。

| # | パターン名 | ストームグラスの状態 | 発動条件 | 具材の動き |
|---|---|---|---|---|
| 1 | 澄み型 | 液体が澄んでいる | 晴れ（WMO 0-1） | 具材はほぼ丼に収まり静止。少数がキラキラとゆっくり漂う。背景クリア |
| 2 | 白濁型 | 液体が白濁する | 寒い日（5℃未満）・雪（71-77, 85-86） | 画面がミルク色の霧に包まれ、米粒が雪のように舞い降りる |
| 3 | 浮遊型 | 細かい具材が浮く | 雨（51-67, 80-82）・霧（45, 48） | 肉・米・玉ねぎが中層〜上層にふわふわ浮遊。雨が強いほど数が増える |
| 4 | 沈殿型 | 下に具材が沈む | 暑い日（30℃以上・晴天寄り） | 具材が丼にどっしり沈殿・堆積。湯気が立ちのぼる |
| 5 | 結晶型 | 上部に結晶風の散らばり | 嵐・雷雨（95-99） | 上部に雪の結晶状の枝が発生し、全体が激しく散らばる（大荒れ牛丼） |
| 6 | 枝状型 | 筋状・枝状に広がる | 強い風（風速8m/s以上、天候問わず優先度低め） | 具材が筋状・枝状の流線に沿って横方向にたなびく |

条件が複合する場合の優先順位: 結晶型 ＞ 白濁型 ＞ 枝状型 ＞ 浮遊型 ＞ 沈殿型 ＞ 澄み型

### 4-3. 気温 → 米の量（ストームグラスの結晶原理）

本物と同じく「寒いほど結晶（＝米）が増える」。パターンとは独立して具材総量に効く。

| 気温 | riceAmount | 見た目 |
|---|---|---|
| 30℃以上 | 0.15 | 米少なめ・澄んだ雰囲気 |
| 20〜30℃ | 0.4 | ほどよい盛り |
| 10〜20℃ | 0.65 | しっかり盛り |
| 0〜10℃ | 0.85 | 大盛り |
| 0℃未満 | 1.0 | 特盛。結晶のようにびっしり（霜の表現） |

※境界はなめらかに補間（20℃と21℃で急に変わらない）

### 4-4. 牛丼パラメータ（内部値）

パターンとテーブルから算出する0.0〜1.0の値。

| パラメータ | 意味 |
|---|---|
| `riceAmount` | 米粒の数（気温連動） |
| `meatAmount` | 肉片の数（荒天ほど増：晴0.5/曇0.6/雨0.7/雪0.6/雷雨1.0） |
| `scatter` | 散らばりの激しさ |
| `turbidity` | 白濁度（画面のミルク色フォグ濃度） |
| `drift` | 横流れ・枝状の強さ |
| `steam` | 湯気の量（暑い日・沈殿型で最大） |

### 4-5. 具材の種類（UI指示書Bセクション）

米粒・牛肉片・玉ねぎ・つゆ滴・紅しょうが・湯気・光の粒・卵（丼の中央、生卵/温玉切替可）

## 5. 画面仕様（UI指示書準拠）

### 5-1. メイン画面レイアウト

```
┌─────────────────────┐
│ 11:42          ☂ 雨  │ ← 上部: 時計(大)・天気アイコン・更新時刻
│ 7月8日 水曜日          │ ← 日付・アプリ名
│                      │
│   🍚  🥩   🧅        │ ← 中央: 具材が浮遊・拡散する空間
│  🥩  💧  🍚  🌶       │    （背景=天気・時間帯の空）
│    🍚   🥩           │
│   ／￣￣￣＼          │
│  ｜ 牛丼＋卵 ｜        │ ← 下部: 丼（常に下部配置・湯気・温かい光）
│   ＼＿＿＿／          │
├─────────────────────┤
│ Yokohama  ☂雨 降雨80% │ ← 情報パネル: 地点/気温/体感/
│ 29℃ Feels like 31℃   │    天気/降水確率/湿度/風速
│ 湿度72% / 風速4.5m/s   │
│ Storm:牛丼乱気流       │ ← 現在のストーム名・パターン名
│ Pattern:浮遊型 ▓▓▓▓░  │
├─────────────────────┤
│[待受][天気更新][パターン]│ ← 操作ボタン5つ
│      [戻す][卵チェンジ] │
└─────────────────────┘
```

### 5-2. 操作ボタン（5つ・UI指示書Eセクション）

| ボタン | 動作 |
|---|---|
| 待受モード | 最小UI表示に切替（時計＋牛丼のみ。ボタン・情報パネル非表示。タップで復帰） |
| 天気更新 | 最新の天気を取得して演出を更新。**鑑賞モード中に押すと実際の天気に復帰** |
| パターン変更 | 鑑賞モード: タップごとに6パターンを順送りで切替（天気連動を一時停止） |
| 戻す | ひとつ前のパターン状態に戻す |
| 卵チェンジ | 丼中央の卵を「生卵 ⇔ 温玉」で切替 |

鑑賞モードの仕様: パターン変更を押した時点で自動連動を一時停止。「天気更新」を押すか、次回の自動更新（30分ごと）が走ると実際の天気に復帰する。鑑賞モード中は画面に小さく「鑑賞モード」と表示して状態を明示する。

### 5-3. 背景演出（UI指示書Cセクション）

背景 = 時間帯 × 天気 の組み合わせで変化。

- 時間帯（端末時刻から判定）: 朝（5-10時）／昼（10-16時）／夕方（16-19時）／夜（19-5時）
- 天気バリエーション: 晴れ／曇り／雨／強風／暑い日／寒い日／雪
- 実装: グラデーション空＋遠景シルエット（夜は夜景ビル・星）＋天候パーティクル（雨筋・雪）で表現

### 5-4. その他の状態（UI指示書Fセクション）

| 状態 | 表示 |
|---|---|
| ローディング中 | 空の丼＋スピナー「天気を取得中… 少々お待ちください」 |
| エラー時 | 丼＋「！」アイコン「通信に失敗しました もう一度お試しください」（前回値があれば背後で表示継続） |

### 5-5. デザイン原則（UI指示書G/Hセクション）

- 3DCGのリアルな質感で具材の細かな動きを美しく表現
- 丼は常に下部配置。温かみのある光で食欲と安心感を演出
- 一見「何のアプリかわからない」が、美しくてずっと眺めてしまう没入感
- 背景: 深い紺の夜空基調（有機ELの焼き付き・電池にも優しい）

### 5-6. PWA

- `manifest.json`: アプリ名「ストーム牛丼」、display: fullscreen、縦固定
- Service Workerでオフライン時も前回の状態を表示

## 6. データ構造

外部保存なし。ブラウザ内変数のみ（卵の設定と待受モードのみlocalStorageに記憶）。

```
weatherState = {
  code, temperature, apparentTemp, windSpeed, humidity,
  precipProbability, fetchedAt, place  // "Yokohama" 等
}

appState = {
  pattern: "浮遊型",        // 現在のパターン（6種）
  viewMode: "auto"|"manual", // 鑑賞モードか
  patternHistory: [...],     // 「戻す」用の履歴
  eggMode: "raw"|"onsen",    // 卵チェンジ
  minimalUI: false           // 待受モード
}

bowlParams = { riceAmount, meatAmount, scatter, turbidity, drift, steam }
```

## 8. 設定項目一覧（コード冒頭のCONFIGに集約）

| 設定名 | 初期値 | 変えると |
|---|---|---|
| `DEFAULT_LOCATION` | 東京(35.68, 139.77) | 位置情報拒否時の地点 |
| `UPDATE_INTERVAL_MIN` | 30 | 天気の自動更新間隔（分） |
| `RICE_MAX_COUNT` / `MEAT_MAX_COUNT` | 400 / 40 | 粒の最大数（重い端末は減らす） |
| `INGREDIENTS` | 米/肉/玉ねぎ/つゆ滴/紅しょうが/光の粒 | 具材の種類・色・サイズ・最大数 |
| `TEMP_TABLE` | 4-3の表 | 気温→米量の対応 |
| `PATTERN_TABLE` | 4-2の表 | 天気→パターンの対応・各パターンの強さ |
| `PATTERN_PRIORITY` | 結晶>白濁>枝状>浮遊>沈殿>澄み | 複合時の優先順位 |
| `BACKGROUNDS` | 時間帯×天気の色定義 | 背景の見た目 |
| `STORM_NAMES` | パターン別の表示名（"牛丼乱気流"等） | 画面下部のStorm表示 |
| `HOT_DAY_TEMP` / `COLD_DAY_TEMP` | 30 / 5 | 暑い日・寒い日の判定しきい値（℃） |
| `WIND_BRANCH_SPEED` | 8 | 枝状型になる風速（m/s） |
| `SHOW_CLOCK` | true | 時計表示の初期状態 |
| `LOW_SPEC_MODE` | false | trueで描画を簡易化（古い端末向け） |

## 9. エラー時の動作

| 異常ケース | 動作 |
|---|---|
| 位置情報を拒否された | 東京の天気で表示。地点名に「東京(デフォルト)」と明示 |
| 天気API失敗（圏外など） | 前回取得値で継続＋「通信に失敗しました もう一度お試しください」。5分後に再試行 |
| 地名の逆引きだけ失敗 | 天気は表示し、地点名は「現在地」とだけ表示 |
| 初回起動でAPIも失敗 | 「晴れ・20℃」のデモ表示＋エラーメッセージ |
| WebGL非対応の古い端末 | 「お使いの端末では表示できません」と案内 |
| Wake Lock非対応 | 静かに無視（スリープ防止だけ効かない） |
