# storm-gyudon（ストーム牛丼グラス）

ストームグラス（気温で結晶が増減する天気占いガラス）を牛丼で再解釈した待受風Webアプリ。

画面下部に丼が鎮座し、現在地の天気・気温に合わせて具材（肉・米・玉ねぎ等）が
浮遊・沈殿・白濁・枝状拡散します。スマホでフルスクリーン表示して待受として使えます。

- 3DCG: Three.js r128（CDN読込・ビルド不要の単一HTML構成）
- 天気: Open-Meteo API（無料・APIキー不要）
- 公開: GitHub Pages（サーバー・DB・アカウント登録は一切不要）
- 仕様書: `docs/` 配下（設計書・AI引き継ぎ仕様書）

## 使い方

| ボタン | 動作 |
|---|---|
| 待受モード | 時計＋牛丼のみの最小表示に切替（画面タップで復帰） |
| 天気更新 | 最新の天気を取得して演出を更新（鑑賞モードも解除） |
| パターン変更 | 6パターンを順送りで鑑賞（天気連動を一時停止・「鑑賞モード」表示） |
| 戻す | ひとつ前のパターンに戻す |
| 卵チェンジ | 丼中央の卵を「生卵 ⇔ 温玉」で切替（次回起動時も維持） |

- 天気は30分ごとに自動更新されます（鑑賞モード中でも自動更新で実際の天気に復帰）
- 位置情報を拒否した場合は「東京(デフォルト)」の天気で表示します
- 圏外・通信失敗時は前回の状態で表示を続け、5分ごとに再試行します

## GitHub Pages 公開手順

1. このリポジトリをGitHubへpushする
2. GitHubのリポジトリページ → **Settings → Pages** を開く
3. **Source: Deploy from a branch**、**Branch: main / (root)** を選んで **Save**
4. 数分待つと `https://<ユーザー名>.github.io/storm-gyudon/` で公開される

※ 位置情報（Geolocation）とService WorkerはHTTPS必須のため、動作確認は
GitHub Pages上か `localhost`（例: `python3 -m http.server`）で行ってください。
file:// では位置情報が動きません。

## スマホのホーム画面に追加（待受として使う）

- **iPhone (Safari)**: 公開URLを開く → 共有ボタン → 「ホーム画面に追加」
- **Android (Chrome)**: 公開URLを開く → メニュー（︙） → 「ホーム画面に追加」

ホーム画面から起動するとフルスクリーン（時計・丼のみの没入表示）になります。

### スリープ防止（Wake Lock）について

画面の消灯を防ぐWake Lock APIを使用していますが、**非対応の端末・ブラウザでは
効きません**（その場合もアプリ自体は正常に動作します）。効かない場合は端末の
画面設定で自動ロックまでの時間を延ばしてください。

## 調整したいとき

`index.html` 冒頭の `CONFIG` にすべての設定（デフォルト地点・更新間隔・具材・
気温テーブル・パターン条件・背景色など）が集約されています。詳細は
`docs/ストーム牛丼グラス_設計書.md` 8章を参照してください。

## 更新時の注意（重要）

`index.html` を更新したら、`sw.js` の `CACHE_VERSION` を必ず上げてください。
上げないと利用者に古いバージョンが配信され続けます。
